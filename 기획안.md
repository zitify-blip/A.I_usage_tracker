# A.I. Usage Tracker 기획안

> 버전: v2.13.0 기준 · 최종 수정: 2026-04-29

---

## 1. 개요

**A.I. Usage Tracker**는 Claude · Gemini · OpenAI · Grok 등 여러 AI 서비스의 사용량과 비용을 한 화면에서 실시간으로 추적하는 Windows 데스크톱 앱입니다. CLI 도구와 API 키 사용량을 동시에 집계하여, AI를 업무·개발에 적극적으로 활용하는 사용자가 비용과 한도를 투명하게 관리할 수 있도록 돕습니다.

### 한 줄 요약

> "Claude · Gemini · OpenAI · Grok 사용량과 비용을 실시간으로 모니터링하는 올인원 데스크톱 앱"

---

## 2. 문제 정의

- AI 서비스가 늘어남에 따라 **각 서비스별로 따로 콘솔에 들어가** 사용량과 비용을 확인해야 함
- Claude CLI · Codex · Grok CLI 같은 **터미널 도구의 누적 사용량**을 한눈에 보기 어려움
- 다중 계정·프로젝트를 운영하는 경우 **계정별 사용량 비교**가 사실상 불가능
- 5시간 세션, 일/주/월 한도 등 **실시간 한도 추적**이 공식 도구만으로는 부족

---

## 3. 타깃 사용자

| 사용자 유형 | 주요 사용 시나리오 |
| --- | --- |
| AI 활용 개발자 | Claude Code · Codex · Grok CLI 사용량과 API 비용을 동시에 추적 |
| 프롬프트 엔지니어 / AI 컨설턴트 | 클라이언트별 API 키를 분리 관리하며 비용 정산 |
| 스타트업 팀 | 팀의 AI 도구 비용을 정기적으로 점검 |
| 헤비 AI 사용자 | Claude Pro/Max의 5시간 세션 한도를 실시간으로 관찰 |

## 4. 핵심 기능

### 4.1 통합 대시보드 (대시보드 탭)
- 모든 서비스의 누적 비용·토큰 합계를 한 카드에 요약
- 일/주/월 단위 토글
- 자동 업데이트 · 인앱 업데이트 알림 버튼

### 4.2 서비스별 탭 (v2.12.2 기준 8개)

| # | 탭 | 내용 |
| --- | --- | --- |
| 0 | 대시보드 | 전체 통합 대시보드 (구 Global) |
| 1 | Claude CLI | claude.ai 세션 기반 5시간 윈도우, 주간 한도 추적 |
| 2 | Claude API | Anthropic Admin API 기반 모델별 토큰·비용 집계 |
| 3 | Gemini API | Google AI Studio API 키 사용량 + Gemini Local Relay |
| 4 | OpenAI API | OpenAI Admin API (`/v1/organization/usage`) |
| 5 | OpenAI CLI | Codex CLI (`~/.codex/sessions`) 로그 분석 |
| 6 | Grok API | xAI `/v1/api-key` 키 정보 조회 |
| 7 | Grok CLI | 커뮤니티 Grok CLI 로컬 로그 best-effort 스캔 |

### 4.3 다중 계정 관리
- 서비스별로 여러 API 키 등록·전환
- **Windows DPAPI**(CurrentUser scope)로 키 암호화 저장
- 표시 시 마스킹 처리 (`sk-...XYZ`)

### 4.4 실시간 비용 계산
- 모델별 단가 테이블 내장 (Claude · Gemini · OpenAI · Grok 주요 모델)
- 입력/출력/캐시 토큰을 분리 집계하여 정확한 비용 산출

### 4.5 Gemini Local Relay
- 로컬 HTTP 프록시 서버를 띄워 외부 SDK·스크립트(Python, curl 등)의 Gemini API 호출을 가로채 사용량 집계
- 앱 내에서 환경변수(`GOOGLE_GENAI_BASE_URL` 등) 원클릭 적용/복원 — 현재 셸 세션에 즉시 반영
- Claude Code용 Gemini 위임 규칙(`~/.claude/CLAUDE.md`)도 원클릭 설치
- Relay 설정 화면은 팝업이 아닌 인라인 페이지로 전환 (v2.12.0)
- 최근 캡처된 요청에 날짜 포함 표시 (v2.12.2)

### 4.6 5시간 세션 추적 (Claude CLI)
- 첫 메시지 시각부터 5시간 윈도우 자동 계산
- 진행률 바(좌→우 채움) 시각화
- 세션 리셋 팝업 제거 — 탭 내 상태 표시로 대체 (v2.12.2)

### 4.7 시스템 통합
- **커스텀 타이틀 바** — WPF 기본 프레임 제거, 앱 전용 상단 바 (v2.12.0)
- **트레이 아이콘** + 풍선 알림 (사용량 임계치 도달 시)
- **단일 인스턴스 보장** (Named Mutex로 중복 실행 방지)
- **자동 업데이트** — GitHub Releases 폴링 → SHA256 검증 → 자동 설치

## 5. 기술 스택

| 영역 | 사용 기술 |
| --- | --- |
| 프레임워크 | WPF · .NET 8 (`net8.0-windows`) |
| 임베디드 브라우저 | Microsoft.Web.WebView2 (Claude.ai 세션 공유) |
| 직렬화 | System.Text.Json |
| 보안 | Windows DPAPI (CurrentUser) |
| 배포 | Self-contained single-file (~160 MB) · Inno Setup 6 · MSIX (선택) |
| 업데이트 채널 | GitHub Releases API |
| 빌드 자동화 | `dotnet publish` + `ISCC.exe` + PowerShell 스크립트 |

## 6. 사용자 경험 (UX) 원칙

1. **한 화면 원칙** — 가장 자주 보는 정보는 대시보드 탭에서 즉시 보이도록
2. **서비스 그룹화** — 같은 서비스의 CLI/API 탭을 인접 배치 (v2.8.2)
3. **다크 테마 단일 적용** — 장시간 모니터링에 부담 없는 톤
4. **백그라운드 우선** — 트레이 상주 + 5초 단위 폴링, 창은 닫아도 동작 유지
5. **마찰 없는 업데이트** — 새 버전 감지 시 클릭 한 번으로 설치
6. **팝업 최소화** — 설정·안내는 인라인 페이지로 전환, 팝업은 필수 확인에만 (v2.12.0)

## 8. 배포

- **GitHub Releases**: <https://github.com/zitify-blip/A.I_usage_tracker/releases/latest>
- **설치 방식**: Inno Setup 인스톨러 (사용자 권한, 관리자 권한 불필요)
- **설치 경로**: `%LOCALAPPDATA%\Programs\ClaudeUsageTracker\` (구 폴더명 유지)
- **자동 시작**: 설치 시 옵션 (Windows 시작 시 자동 실행)
- **무결성 검증**: 모든 릴리즈에 `.sha256` 사이드카 첨부 → 자동 업데이트 시 검증
